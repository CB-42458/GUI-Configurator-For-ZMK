ZMK.Config

This module contains classes and functions which allow with given data to generate files within a directory structure which will be pushed to GitHub that will leverage GitHub actions to build a firmware for a keyboard.

for the type hints in methods which are either not implemented or partially implemented "classmethod" is used to represent a class as I couldn't think of a better way to do that.

 30        """
 31        Constructor for the ZMKConfig class creates all the attributes which are used in a ZMK configuration.
 32        
 33        **Class Attributes**  
 34        `__config_name` : str | None - Name of the keyboard configuration
 35        
 36        `__config_id` : str | None - ID of the keyboard configuration, used by ZMK to identify the keyboard
 37        
 38        `__shield_directory` : str | None - Directory of the shield used by the keyboard found in the app/boards/shields
 39        directory of the ZMK repo
 40        
 41        `__working_directory` : str | None - Directory where the configuration will be saved locally
 42        
 43        `__features` : list[Features.AbstractFeature] - Features that the keyboard will have, such as rotary encoder,
 44        RGB under-glow, etc.
 45        
 46        `__mcu` : MCUs.AbstractMCU | None - MCU that the keyboard will use
 47        
 48        `__default_config_options` : list - Default configuration options that will be used by the keyboard, outputted
 49        in the .defconfig file
 50        
 51        `__config_options` : list - Configuration options that will be used by the keyboard, outputted in the .conf
 52        file
 53        
 54        `__driver` : Drivers.AbstractDriver | None - Driver that the keyboard will use such as a matrix driver, demux
 55        driver, etc.
 56        
 57        `__split_config` : bool - Boolean value which represents if the keyboard is a split keyboard or not
 58        
 59        `__split_config_options` : TypedDict('__split_config_options', {'central': list, 'peripheral': list}) -
 60        Configuration options for the respected halves of the split keyboard
 61        
 62        `__behaviours` : list[Behaviours.AbstractBehaviour] - List will store behaviors which are unique configurations
 63        of a behaviour such as a tap dance
 64        
 65        `__keymap` : CusDataStruc.Array - Array which will store behaviours for the keymap
 66        
 67        `__transform` : Transform.MatrixTransform | None - Transform which is used to translate the physical layout of
 68        the keyboard to the logical layout
 69        """
 70        self.__config_name: str | None = None
 71        self.__config_id: str | None = None
 72        self.__shield_directory: str | None = None
 73        self.__working_directory: str | None = None

 75        self.__mcu: MCUs.AbstractMCU | None = None
 76        self.__default_config_options: list = []
 77        self.__config_options: list = []
 78        self.__driver: Drivers.AbstractDriver | None = None

 95        """
 96        Method will save the configuration to a json file. If overwrite is not True and the file already exists then

314    def add_split_config_option(self, side: str, option: ConfigOptions.AbstractConfigOption) -> None:
315        """
316        Method for adding a split config option.
317
318        @param side: String which is either 'Primary' or 'Secondary'  
319        @param option: ConfigOption which is added to the respected side
320        """
321        if side in self.__split_config_options.keys():
322            # noinspection PyTypedDict
323            self.__split_config_options[side].append(option)
324        else:
325            raise ValueError(
326                f"side: {side} is not valid side. Valid sides {[key for key in self.__split_config_options]}")
327
328    def del_split_config_option(self, side: str, option: classmethod) -> None:
329        """
330        Method for deleting a split config option.
331
332        @param side: String which is either 'Primary' or 'Secondary'  
333        @param option: ConfigOption which is deleted from the respected side
334        """
335        if side in self.__split_config_options:
336            # noinspection PyTypedDict
337            self.__split_config_options[side].remove(option)
338        else:
339            raise ValueError(
340                f"side: {side} is not valid side. Valid sides {[key for key in self.__split_config_options]}")
